在28台PC集群上搭建HADOOP 3.0 集群

在Apache官网上可下载 Hadoop 3.0 GA版，有编译好的二进制版本，另外也附有源码，本帖重点介绍安装过程，故直接使用编译好的二进制版本进行安装，后续文章将继续分析及改写源码。

1.系统安装ubuntu 16.04 64 位（28台）

建议在/etc/hostname 下修改节点名称，如nodei(i=1..28)，在/etc/hosts 下配置各节点与内网IP的对应，插入如下代码（IP自行定义，建议设置为连续值，方便管理）：

1.1修改系统的源

为简化操作，提供两种方法替换源：

1.使用 snullp 大叔开发的 配置生成器替换掉/etc/apt/sources.list下的sources.list。先下载sources.list（ubuntu）版,然后执行：

$sudo sed -i 's/archive.ubuntu.com/mirrors.ustc.edu.cn/g' /etc/apt/sources.list
$sudo apt-get update

2.直接使用命令

$sudo cp sosources.list /etc/apt/sources.list 
$sudo apt-get update

1.2记录每台的Mac地址

记录Mac地址的目的是为了之后在路由上配置静态IP与Mac地址绑定，使用ifconfig即可查看内网IP与Mac地址，此处建议用文档拷贝下来。

2.ssh 服务配置

2.1 安装ssh服务

直接使用apt-get install

$sudo apt-get install ssh

2.2 配置PC间的无密访问

在.ssh目录下执行命令：

$ssh-keygen -t rsa

连续四下回车，生成id_rsa（私钥）、id_ras.pub（公钥）

给自己主机设置无密访问：

$ssh-copy-id nodei #i为本地主机的序号

配置集群间无密访问：

$scp ~/.ssh/authorized_keys hadoop@node1:~/.ssh/nodei/i #i为本地主机的序号

以上均需在每台PC上执行，然后在node1的~/.ssh/目录下就有了2..28节点的密码值authorized_keys，然后在命令行执行：

$for i in {2..28};do cat $i >> authorized_keys; done

即把每个节点密码值都追加到node1上的authorized_keys， 最后再把所有节点的密码文件authorized_keys发送到各个节点，使用脚本：

$for i in {2..28};do scp ~/.ssh/authorized_keys hadoop@node$i:~/.ssh/;done

至此已完成各节点间hadoop用户下的无密访问。为了有时能够执行root权限脚本，比如用脚本给集群关机、重启，给系统文件夹写文件等，可在root用户下重复以上操作（建议操作）。

4.Java 安装

Hadoop 3.0需要JDK 1.8+（源码BUILDING文件已强调），建议使用Oracle JDK1.8+。

4.1 Oracle JAVA安装

在Oracle官网上下载JDK，SE版本即可，本文示例为jdk-8u151-linux-x64.tar.gz。步骤如下：

$sudo mkdir /usr/lib/jvm #在系统文件夹下新建jvm文件夹
$tar zxvf jdk-8u151-linux-x64.tar.gz -C /usr/lib/jvm/ #解压

4.2 环境变量配置

打开系统环境变量文件/etc/profile

$sudo vim /etc/profile

插入下面几行

#set JAVA environment
JAVA_HOME=/usr/lib/jvm/jdk1.8.0_151
PATH=$JAVA_HOME/bin:$PATH
CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
export JAVA_HOME
export PATH

最后source一下：

$source /etc/profie

5.Hadoop 3.0安装及配置

5.1 Hadoop 安装

在~目录下解压hadoop-3.0.0.tar.gz，使用编译好的二进制版本进行安装：

$tar zxvf hadoop-3.0.0.tar.gz 
$mv hadoop-3.0.0 ~/hadoop

如4.2，配置下Hadoop的环境变量：
/etc/profile插入下面几行：

export HADOOP_HOME=/home/hadoop/hadoop
export PATH=$PATH:$HADOOP_HOME/sbin
export PATH=$PATH:$HADOOP_HOME/binexport HADOOP_HOME=/home/hadoop/hadoop
export PATH=$PATH:$HADOOP_HOME/sbin
export PATH=$PATH:$HADOOP_HOME/bin

最后source一下：

$source /etc/profie

5.2 Hadoop 参数配置

参数配置主要有下面几个文件： 

#hadoop-env.sh   
export JAVA_HOME=/usr/lib/jvm/jdk1.8.0_151

#core-site.xml
<configuration>
<property>  
    <name>fs.defaultFS</name>  
    <value>hdfs://node1:/</value> 
</property>
</configuration>

#hdfs-site.xml
<configuration>
<property>  
<name>dfs.replication</name>  
<value>3</value>  
</property>
<property>
      <name>dfs.permissions</name>
      <value>false</value>
</property> 
</configuration> 

#yarn-site.xml
<configuration>
 <property>
<name>yarn.resourcemanager.hostname</name>
   <value>node1</value>
 </property>
 <property>
   <name>yarn.nodemanager.aux-services</name>
   <value>mapreduce_shuffle</value>
 </property>
<!-- Site specific YARN configuration properties -->
</configuration>

用脚本发送到每个节点：

$for i in {2..28};do scp -r ~/hadoop hadoop@node$i:~/;done

在开启之前，建议重启下机器，脚本如下（已配置好root用户下各节点点无密访问）：

#!/bin/sh
for i in $(seq 2 28)
do
ssh root@node$i "shutdown -r 1"
done
reboot now

开启Hadoop 3.0集群

$hdfs namenode -format #格式化namenode
$start-dfs.sh #开启HDFS
$start-yarn.sh #开启MapReduce
$netstat -lnt #查看9870端口是否开启
